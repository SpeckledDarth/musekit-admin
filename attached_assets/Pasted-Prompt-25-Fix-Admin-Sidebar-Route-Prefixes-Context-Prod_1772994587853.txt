Prompt 25: Fix Admin Sidebar Route Prefixes
Context
Production Target
Next.js 14.2.18 (App Router) + Supabase + Vercel
This module lives at packages/admin/ in the monorepo
All admin pages are routed under /admin/ in the Next.js App Router (e.g., /admin/users, /admin/setup/branding)
Problem
Every sidebar href in AdminSidebar.tsx and SetupSidebar.tsx is missing the /admin prefix. The sidebar uses paths like /users, /setup, /settings — but the actual Next.js routes are at /admin/users, /admin/setup, /admin/settings.

What happens:

Clicking "Setup" navigates to /setup/branding → 404 error
Clicking "Users" navigates to /users → caught by CMS [slug] catch-all, shows blank CMS page instead of admin users page
Same problem for every other sidebar link: Metrics, Audit Log, Revenue, Feature Toggles, Customer Service, Onboarding, Settings
The "Back" link in the drill-in view (Setup sub-pages) also uses href="/" instead of href="/admin"
Impact
Every single navigation link in the admin sidebar is broken. Users can only reach admin pages by typing URLs directly.
