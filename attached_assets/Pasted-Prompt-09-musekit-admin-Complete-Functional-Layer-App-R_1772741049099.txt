Audit Logging (applies to ALL CRUD operations)

Admin Sidebar Redesign — Vercel-Style Drill-In Navigation
The current flat sidebar must be replaced with a Vercel-style drill-in navigation pattern. This is a core UX pattern used across both the admin dashboard and (later) the user dashboard.

Navigation Pattern
Primary view: Shows only top-level nav items. Items that contain sub-pages display a > (ChevronRight) icon on the right side of the row.

Sub-menu view: When a top-level item with children is clicked, the entire sidebar content is replaced with:

A back row at the top: ← Section Name (ArrowLeft icon + label) — clicking returns to the primary view
The sub-items listed below, each as a normal nav link
Auto-detection: The sidebar reads usePathname() on mount. If the current URL matches a sub-section prefix (e.g., /admin/setup/branding starts with /admin/setup), auto-expand that section's sub-menu. This ensures deep-links and page refreshes show the correct sub-menu.

Search: A search/filter input at the top of the sidebar filters the currently visible items (top-level or sub-menu items). Use lucide-react Search icon. Filter is client-side, case-insensitive, matching on label text.

Component Architecture
Refactor AdminSidebar (src/layout/AdminSidebar.tsx):

interface NavItem {
  label: string;
  href: string;
  icon: LucideIcon;
}
interface SubMenu {
  label: string;
  icon: LucideIcon;
  prefix: string; // URL prefix for auto-detection (e.g., "/admin/setup")
  items: NavItem[];
}
interface NavConfig {
  topLevel: NavItem[];
  subMenus: SubMenu[];
  bottom: NavItem[];
}
State management:

activeSubMenu: string | null — the key of the currently open sub-menu, or null for primary view
Derived from usePathname() on mount: loop through subMenus, check if pathname.startsWith(subMenu.prefix) — if so, set activeSubMenu to that key
Clicking a parent item with children sets activeSubMenu; clicking the back arrow clears it
Rendering logic:

if (activeSubMenu) {
  render: ← back row (← {subMenu.label})
  render: subMenu.items (filtered by search)
} else {
  render: topLevel items (filtered by search)
  render: subMenu parent items with > chevron (filtered by search)
  render: --- divider ---
  render: bottom items
}
Active item highlighting:

Exact match (pathname === item.href) or prefix match (pathname.startsWith(item.href + "/")) — same as current
For parent items in primary view: highlight if any child route matches the current pathname
Collapsed mode:

When sidebar is collapsed (icons only), clicking an icon that has children opens a popover/tooltip with the sub-menu items
Back arrow is not needed in collapsed mode — just show the parent section icon
Styling:

Match existing dark theme: bg-card, border-r, text colors from current theme
Back row: text-muted-foreground hover:text-foreground, font-medium, with ArrowLeft icon
