Module Repl

Context
The admin package renders 28+ pages including dashboards, user management, affiliate management, setup wizards, and data visualizations. Currently, pages use hardcoded Tailwind grays and charts use hardcoded hex/HSL colors. After Prompt 38 (ThemeInjector) and Prompt 39 (shared Tailwind preset), all CSS variables and Tailwind mappings are available. This prompt migrates every admin page to consume them.

Reference: See docs/current/css-variable-contract.md for the complete list of available CSS variables, their types, defaults, and recommended Tailwind class replacements.

The Fix
1. Update tailwind.config.ts
Import the shared preset from @musekit/design-system and extend from it:

import { tailwindPreset } from "@musekit/design-system";
const config = {
  presets: [tailwindPreset],
  content: ["./src/**/*.{js,ts,jsx,tsx}"],
  // ... keep existing plugins
};
This gives the admin package access to all semantic colors, chart colors, fonts, shadows, and radius mappings.

2. Migrate Layout Components
Replace all hardcoded grays in:

src/layout/AdminHeader.tsx
src/layout/AdminSidebar.tsx
src/layout/AdminLayout.tsx
src/layout/Breadcrumb.tsx
Use the replacement rules from the CSS Variable Contract (see docs/current/css-variable-contract.md):

bg-white → bg-background
bg-gray-* → bg-muted or bg-accent
text-gray-* → text-foreground or text-muted-foreground
border-gray-* → border-border
dark:bg-gray-* → dark:bg-background or dark:bg-card
