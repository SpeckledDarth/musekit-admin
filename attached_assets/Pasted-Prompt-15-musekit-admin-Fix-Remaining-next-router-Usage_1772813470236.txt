Prompt 15: musekit-admin — Fix Remaining next/router Usage (8 Files)
Module Repl
musekit-admin (GitHub: SpeckledDarth/musekit-admin)

Context
The AdminSidebar was successfully converted to next/navigation, but 8 other files still use import { useRouter } from "next/router". This crashes every admin route in the App Router integration monorepo with "NextRouter was not mounted."

This is a small, focused fix. Do NOT change any logic, UI, or functionality — only change the router imports and API calls.

The Fix
Replace next/router with next/navigation equivalents in all 8 files below. The key API differences are:

Pages Router (next/router)	App Router (next/navigation)
import { useRouter } from "next/router"	import { useRouter, usePathname, useParams, useSearchParams } from "next/navigation"
router.pathname	usePathname() (call at component top level)
router.query (for route params like [id])	useParams() (call at component top level)
router.query (for query string like ?search=foo)	useSearchParams() (call at component top level)
router.push(path)	router.push(path) (same API)
router.replace(path)	router.replace(path) (same API)
router.replace({ pathname, query })	router.replace(pathname + "?" + new URLSearchParams(query).toString())
router.isReady	Not needed — always ready in App Router
Important: Every file using next/navigation hooks must have "use client" at the top.

3. src/hooks/useListView.ts (line 2)
Current usage:

import { useRouter } from "next/router";

const query = router.query;
// ...
for (const [k, v] of Object.entries(router.query)) { ... }
router.replace({ pathname: router.pathname, query: newQuery }, undefined, { shallow: true });
Fix: This is the most complex conversion. Replace with:

import { usePathname, useSearchParams, useRouter } from "next/navigation";
const router = useRouter();
const pathname = usePathname();
const searchParams = useSearchParams();
// Convert searchParams to a plain object where router.query was used:
const query = Object.fromEntries(searchParams.entries());
// For router.replace with query:
const newParams = new URLSearchParams();
for (const [k, v] of Object.entries(newQuery)) {
  if (v !== undefined && v !== null && v !== "") {
    newParams.set(k, String(v));
  }
}
router.replace(`${pathname}?${newParams.toString()}`);
Note: The { shallow: true } option does not exist in App Router's router.replace() — remove it. App Router navigation is shallow by default for client-side transitions.
