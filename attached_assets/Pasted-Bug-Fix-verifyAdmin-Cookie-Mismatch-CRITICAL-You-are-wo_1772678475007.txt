Bug Fix: verifyAdmin() Cookie Mismatch — CRITICAL
You are working in the musekit-admin Repl, which contains the @musekit/admin package. This package provides the entire admin dashboard for a SaaS platform — user management, metrics, audit logs, settings, feature toggles, and more.

Stack: React 18, TypeScript, Next.js API routes (Pages Router style), Tailwind CSS v3, @supabase/supabase-js, shadcn/ui components, recharts, lucide-react.

The Problem
Every single admin API route returns 401 Unauthorized for logged-in users. The entire admin dashboard is broken — no data loads, no saves work, nothing functions. This is caused by a single bug in the authentication function.

Root Cause
File: src/lib/admin-auth.ts

The verifyAdmin() function is called by every admin API route to authenticate the user. When no Authorization: Bearer <token> header is present (which is the normal case for browser requests), it falls back to reading a session cookie. The problem is it looks for a cookie named sb-access-token — this cookie does not exist.

Supabase SSR (@supabase/ssr) stores session tokens in cookies named sb-<project-ref>-auth-token, split into chunks like .0, .1, etc. The hardcoded name sb-access-token never matches any cookie, so the function always returns 401.

Current broken code (full file, src/lib/admin-auth.ts):

import { createClient } from "@supabase/supabase-js";

      res.status(401).json({ error: "Unauthorized" });
      return null;                                        // <-- always hits this
    }
    const {
      data: { user },
      error,
    } = await supabase.auth.getUser(sbAccessToken);

      res.status(401).json({ error: "Unauthorized" });
      return null;

      .from("profiles")
      .select("role, email")
      .eq("id", user.id)
      .single();

      !profile ||
      (profile.role !== "admin" && profile.role !== "super_admin")

      res.status(403).json({ error: "Forbidden: Admin access required" });
      return null;
    }
    return { userId: user.id, email: profile.email, role: profile.role };
  }
  // ... Bearer token path (works but never reached from browser)
}
function extractCookieValue(
  cookieHeader: string,
  name: string
): string | null {
  const match = cookieHeader.match(
    new RegExp(`(?:^|;\\s*)${name}=([^;]*)`)
  );
  return match ? decodeURIComponent(match[1]) : null;
}

import type { NextApiRequest, NextApiResponse } from "next";
import { createClient } from "@supabase/supabase-js";
import { createServerClient } from "@supabase/ssr";
const supabaseUrl = process.env.NEXT_PUBLIC_SUPABASE_URL || "";
const supabaseAnonKey = process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY || "";
export async function verifyAdmin(
  req: NextApiRequest,
  res: NextApiResponse
): Promise<{ userId: string; email: string; role: string } | null> {
  const authHeader = req.headers.authorization;
  const token = authHeader?.replace("Bearer ", "");
  let user: { id: string; email?: string; [key: string]: unknown } | null = null;
  if (token) {
    // Bearer token path (for programmatic API calls)
    const supabase = createClient(supabaseUrl, supabaseAnonKey);
    const { data, error } = await supabase.auth.getUser(token);
    if (error || !data.user) {
      res.status(401).json({ error: "Unauthorized" });
      return null;
    }
    user = data.user;
  } else {
    // Cookie path (for browser requests) — use @supabase/ssr
    const supabase = createServerClient(supabaseUrl, supabaseAnonKey, {
      cookies: {
        getAll() {
          const cookieHeader = req.headers.cookie || "";
          return cookieHeader
            .split(";")
            .filter((c) => c.trim())
            .map((c) => {
              const [name, ...rest] = c.trim().split("=");
              return { name, value: rest.join("=") };
            });
        },
        setAll() {
          // Read-only for API route verification
        },
      },
    });
    const { data, error } = await supabase.auth.getUser();
    if (error || !data.user) {
      res.status(401).json({ error: "Unauthorized" });
      return null;
    }
    user = data.user;
  }
  // Check admin role in profiles table
  const { createSupabaseAdmin } = await import("./supabase");
  const adminClient = createSupabaseAdmin();
  const { data: profile } = await adminClient
    .from("profiles")
    .select("role, email")
    .eq("id", user.id)
    .single();
  if (
    !profile ||
    (profile.role !== "admin" && profile.role !== "super_admin")
  ) {
    res.status(403).json({ error: "Forbidden: Admin access required" });
    return null;
  }
  return { userId: user.id, email: profile.email, role: profile.role };
}

Replaced extractCookieValue(cookieHeader, "sb-access-token") with createServerClient from @supabase/ssr
The SSR client automatically finds and reconstructs Supabase auth cookies regardless of project ref or chunking
Uses getUser() (no token argument) — the SSR client handles session resolution internally
Deduplicated the admin role check (was duplicated in both paths)
Removed the extractCookieValue helper (no longer needed)
