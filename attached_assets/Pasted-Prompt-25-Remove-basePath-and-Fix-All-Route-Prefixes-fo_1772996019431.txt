Prompt 25: Remove basePath and Fix All Route Prefixes for Monorepo
Context
Production Target
Next.js 14.2.18 (App Router) + Supabase + Vercel
This module lives at packages/admin/ in a Turborepo monorepo
All admin pages are routed under /admin/ via the App Router directory structure (apps/web/src/app/admin/...)
All admin API routes are at /api/admin/... (apps/web/src/app/api/admin/...)
There is no basePath in the monorepo's Next.js config
Problem: basePath Conflict
This admin module was built as a standalone Repl with basePath: "/admin" in next.config.js. That setting causes Next.js to automatically prepend /admin to all <Link href> and router.push() calls. So the code uses paths like /users and /setup/branding, relying on basePath to produce /admin/users and /admin/setup/branding.

In the monorepo, basePath does not exist. Routes are handled by the App Router directory structure. So:

href="/users" navigates to /users (wrong — should be /admin/users)
href="/setup/branding" navigates to /setup/branding (wrong — should be /admin/setup/branding)
fetch("/admin/api/admin/overview") calls /admin/api/admin/overview (wrong — should be /api/admin/overview)
The /admin prefix in fetch paths was doubled by basePath — the code has /admin/api/admin/... but the actual monorepo API routes are at /api/admin/....

Impact
Every sidebar navigation link is broken — pages either 404 or land on the wrong route
API calls are hitting wrong endpoints — the /admin/api/admin/... prefix is wrong for the monorepo
Fix — Three Categories of Changes
Category 1: Remove basePath from next.config.js
If next.config.js (or next.config.mjs / next.config.ts) contains basePath: "/admin", remove that line. It is not needed in the monorepo — the App Router directory structure handles the /admin prefix.
